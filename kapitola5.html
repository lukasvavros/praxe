Kapitola 5: LCD displej a sběrnice I2C

Praxe – Arduino | 3. ročník

Opakování z třetí lekce – servomotor a potenciometr

V následujícím programu jsou tři chyby. Program má načítat polohu potenciometru, převést ji na úhel od 0° do 180° a podle výsledku nastavovat polohu servomotoru.

Úloha – najděte a opravte chyby v programu

Cíl: Prohlédněte si program, najděte tři chybné řádky a opravte je tak, aby servomotor správně reagoval na otáčení potenciometru. 

#include <Servo.h>

Servo servo1;

const byte pinServo = 9;
const byte pinPotenciometr = A0;

void setup() {
  servo1.attach(pinPotenciometr);
}

void loop() {
  int hodnotaPotenciometru = digitalRead(pinPotenciometr);
  int uhel = map(hodnotaPotenciometru, 0, 1023, 0, 180);

  servo1.write(hodnotaPotenciometru);
  delay(15);
}

Co má program dělat

  1. Servomotor je řídicím vodičem připojený na digitální pin D9.
  2. Potenciometr je připojený ke vstupu A0.
  3. Arduino načítá analogovou hodnotu potenciometru v rozsahu 0 až 1023.
  4. Funkce map() převede naměřenou hodnotu na úhel 0° až 180°.
  5. Servomotor se natočí na vypočítaný úhel.
Nápověda: Zkontrolujte pin ve funkci attach(), způsob čtení potenciometru a hodnotu předanou funkci write().

Cíle kapitoly

V této kapitole připojíme k Arduinu znakový LCD displej s I2C adaptérem. Naučíme se instalovat novou knihovnu, vysvětlíme si komunikaci po sběrnici I2C a zobrazíme na displeji text i hodnoty z programu.

Instalace knihovny Nainstalujeme knihovnu do Arduino IDE ze souboru s příponou .zip.
Sběrnice I2C Vysvětlíme si význam vodičů SDA a SCL a princip adresování zařízení.
Zobrazení dat Na LCD displeji zobrazíme text, odpočet i polohu potenciometru v procentech.

LCD displej 16 × 2

Znakový LCD displej slouží k zobrazování krátkých textů, čísel a naměřených hodnot. Označení 16 × 2 znamená, že displej má 16 znakových pozic v každém řádku a celkem dva řádky.

Samotný LCD displej má mnoho vývodů. I2C adaptér na jeho zadní straně však převádí komunikaci tak, že pro spojení s Arduinem potřebujeme pouze čtyři vodiče: GND, VCC, SDA a SCL.

Displej s I2C adaptérem potřebuje pouze napájení a dva komunikační vodiče.

16 sloupců Na jeden řádek lze zobrazit maximálně 16 běžných znaků.
2 řádky Řádky jsou v programu číslované od nuly: první řádek je 0, druhý řádek je 1.
I2C adresa Často se používá adresa 0x27. U jiného modulu může být adresa například 0x3F.

I2C sběrnice

I2C je zkratka z anglického názvu Inter-Integrated Circuit. Jde o sériovou sběrnici, která umožňuje komunikaci mezi Arduinem a dalšími zařízeními, například senzory, displeji nebo převodníky.

Pro přenos se používají pouze dva signálové vodiče. Všechna zařízení jsou připojena ke stejné dvojici vodičů a rozlišují se pomocí své I2C adresy.

Více zařízení může sdílet stejné vodiče SDA a SCL, pokud mají rozdílné adresy.

SDA – data Přenáší vlastní data a adresy mezi Arduinem a připojenými zařízeními.
SCL – hodiny Určuje časování komunikace, aby všechna zařízení četla data ve správný okamžik.
Adresa zařízení Arduino pomocí adresy vybere zařízení, se kterým chce právě komunikovat.

Výhody I2C

Úspora pinů Pro komunikaci s více zařízeními stačí dva signálové vodiče.
Jednoduché zapojení Méně kabelů znamená přehlednější obvod a menší riziko chyby.
Více zařízení Na jednu sběrnici lze připojit více modulů, pokud se jejich adresy nekříží.
Knihovna Wire Arduino obsahuje knihovnu Wire.h, která zajišťuje základní komunikaci po I2C.
Obousměrný přenos Arduino může podle typu zařízení data odesílat i přijímat.
Rozšiřitelnost Stejnou sběrnici lze později doplnit o další displeje, senzory nebo převodníky.
Doplňující přehled sběrnic: HiBit – UART, I2C and SPI

Zapojení LCD displeje k Arduinu UNO

I2C adaptér displeje má obvykle čtyři vývody. U desky Arduino UNO jsou signály I2C připojené k analogovým pinům A4 a A5.

GND → GND Společná zem LCD displeje a Arduina.
VCC → 5 V Napájecí napětí pro LCD displej a I2C adaptér.
SDA → A4 Datový vodič sběrnice I2C na desce Arduino UNO.
SCL → A5 Hodinový vodič sběrnice I2C na desce Arduino UNO.

Zapojení LCD displeje s I2C adaptérem k desce Arduino UNO.

Pozor: Umístění pinů SDA a SCL se může u jiných desek lišit. U Arduino UNO odpovídají pinům A4 a A5. Vždy zkontrolujte popis konkrétní desky.

Použití knihovny LiquidCrystal_I2C

Nejprve načteme knihovnu a vytvoříme objekt, který bude představovat náš LCD displej. Objekt vytvoříme nad funkcí setup(), aby jej bylo možné používat v celém programu. 

#include <LiquidCrystal_I2C.h>

LiquidCrystal_I2C lcd(0x27, 16, 2);
#include Načte knihovnu LiquidCrystal_I2C a zpřístupní její příkazy.
0x27 I2C adresa displeje. U některých modulů může být jiná.
16, 2 Displej má 16 znakových sloupců a dva řádky.

Základní příkazy

lcd.init(); Inicializuje displej a připraví jej k použití.
lcd.backlight(); Zapne podsvícení LCD displeje.
lcd.clear(); Vymaže všechny znaky z displeje a přesune kurzor na začátek.
lcd.setCursor(0, 1); Nastaví kurzor na první sloupec druhého řádku.
lcd.print("Text"); Vypíše text nebo hodnotu od aktuální pozice kurzoru.
lcd.noBacklight(); Vypne podsvícení, obsah displeje však zůstane zachovaný.
Různé verze knihovny: Některé knihovny používají pro spuštění displeje příkaz lcd.begin(); místo lcd.init();. Použijte příkaz uvedený v příkladech nainstalované knihovny.

Souřadnice displeje

Funkce lcd.setCursor(sloupec, řádek) používá číslování od nuly. První znak má souřadnice (0, 0) a poslední znak druhého řádku má souřadnice (15, 1).

0,0 15,0
0,1 15,1

První program pro LCD displej

Následující program inicializuje displej, zapne podsvícení a zobrazí text na obou řádcích. Protože se text nemění, stačí jej vypsat jednou ve funkci setup(). 

#include <LiquidCrystal_I2C.h>

LiquidCrystal_I2C lcd(0x27, 16, 2);

void setup() {
  lcd.init();
  lcd.backlight();
  lcd.clear();

  lcd.setCursor(0, 0);
  lcd.print("Hello world!");

  lcd.setCursor(0, 1);
  lcd.print("Arduino LCD");
}

void loop() {
  // Text se nemeni, proto zde nemusi byt zadny kod.
}
Zapamatujte si: Pokud se zobrazený text nemění, vypíšeme jej ve funkci setup(). Hodnoty, které se mají průběžně aktualizovat, vypisujeme ve funkci loop().

Praktické úkoly

Úkol 1 – jméno a příjmení

Připojte LCD displej k Arduinu a vytvořte program, který zobrazí vaše jméno na prvním řádku a příjmení na druhém řádku.

  1. Načtěte knihovnu LiquidCrystal_I2C.
  2. Vytvořte objekt displeje s adresou 0x27 a rozměrem 16 × 2.
  3. Ve funkci setup() displej inicializujte a zapněte podsvícení.
  4. Nastavte kurzor na začátek prvního řádku a zobrazte jméno.
  5. Nastavte kurzor na začátek druhého řádku a zobrazte příjmení.
Kontrola výsledku: Oba texty musí zůstat na displeji zobrazené a nesmí se neustále překreslovat nebo blikat.

Úkol 2 – odpočet

Zobrazte na displeji odpočet od 10 do 0. Každé číslo bude zobrazené po dobu 0,5 sekundy. Po skončení odpočtu zobrazte libovolný krátký vzkaz.

  1. Použijte cyklus for s proměnnou, která začíná hodnotou 10.
  2. Po každém průchodu snižte hodnotu proměnné o jedna.
  3. Číslo zobrazujte na stejném místě displeje.
  4. Mezi jednotlivými hodnotami použijte delay(500).
  5. Po dokončení cyklu displej vyčistěte a zobrazte vlastní vzkaz.
Nápověda: Po jednociferném čísle vypište jednu mezeru navíc. Tím odstraníte zbytek předchozí hodnoty 10.

Úkol 3 – poloha potenciometru v procentech

Připojte potenciometr k analogovému vstupu A0 a na LCD průběžně zobrazujte jeho polohu v rozsahu 0 až 100 %.

  1. Jeden krajní vývod potenciometru připojte na 5 V.
  2. Druhý krajní vývod připojte na GND.
  3. Střední vývod – jezdec – připojte na analogový vstup A0.
  4. Funkcí analogRead(A0) načítejte hodnotu v rozsahu 0 až 1023.
  5. Funkcí map() převeďte hodnotu na rozsah 0 až 100.
  6. Na prvním řádku zobrazte text Potenciometr:.
  7. Na druhém řádku zobrazujte vypočítané procento a znak %.
procenta = map(hodnota, 0, 1023, 0, 100)
Rozšiřující úkol: Vytvořte na druhém řádku jednoduchý sloupcový ukazatel. Podle polohy potenciometru zobrazujte postupně více znaků.

Kontrola I2C adresy

Pokud displej po zapojení nereaguje, může používat jinou I2C adresu než 0x27. Pomocí následujícího programu lze vyhledat adresy zařízení připojených ke sběrnici I2C. 

#include <Wire.h>

void setup() {
  Wire.begin();
  Serial.begin(9600);
  Serial.println("Hledam I2C zarizeni...");
}

void loop() {
  byte pocet = 0;

  for (byte adresa = 1; adresa < 127; adresa++) {
    Wire.beginTransmission(adresa);
    byte chyba = Wire.endTransmission();

    if (chyba == 0) {
      Serial.print("Nalezena adresa: 0x");

      if (adresa < 16) {
        Serial.print("0");
      }

      Serial.println(adresa, HEX);
      pocet++;
    }
  }

  if (pocet == 0) {
    Serial.println("Zadne I2C zarizeni nebylo nalezeno.");
  }

  delay(3000);
}
Postup: Nahrajte program do Arduina, otevřete sériový monitor s rychlostí 9600 baudů a nalezenou adresu doplňte do konstruktoru displeje. 
LiquidCrystal_I2C lcd(0x27, 16, 2);

Nejčastější potíže

  1. Displej svítí, ale nejsou vidět znaky: malým šroubovákem upravte kontrast pomocí trimru na I2C adaptéru.
  2. Displej nereaguje: zkontrolujte zapojení vodičů GND, VCC, SDA a SCL.
  3. Program se nahraje, ale nic se nezobrazuje: ověřte správnou I2C adresu pomocí skeneru.
  4. Arduino hlásí chybu u lcd.init(): vyzkoušejte příkaz lcd.begin() podle použité verze knihovny.
  5. Na displeji zůstávají staré znaky: před kratší hodnotou vypište mezery nebo vhodně použijte lcd.clear().
